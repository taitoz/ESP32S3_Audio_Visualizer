# Gear VR Controller - Pairing Guide

## Режим сопряжения (Pairing Mode)

### Как ввести контроллер в режим сопряжения

**Метод 1: Первое включение**
1. Вставьте батарейку CR2032 (если ещё не вставлена)
2. Контроллер автоматически войдёт в режим сопряжения
3. Светодиод будет **мигать синим** (быстро)

**Метод 2: Принудительный сброс**
1. Зажмите **Home** + **Trigger** одновременно
2. Держите **5 секунд**
3. Светодиод начнёт **мигать синим** (режим сопряжения активен)
4. Отпустите кнопки

**Метод 3: Через меню (если контроллер уже был подключен к телефону)**
1. Зажмите **Home** на 2 секунды
2. В меню выберите "Unpair" или "Forget device"
3. Контроллер войдёт в режим сопряжения

---

## Индикация светодиода

| Цвет | Режим | Описание |
|------|-------|----------|
| **Синий (мигает быстро)** | Pairing mode | Ищет новое устройство |
| **Синий (горит)** | Connected | Подключен к устройству |
| **Красный (мигает)** | Low battery | Батарея разряжена (<10%) |
| **Выкл** | Sleep/Off | Контроллер спит или выключен |

## Подключение к ESP32-S3

### Шаг 1: Подготовка контроллера

1. **Включите контроллер** (нажмите любую кнопку)
2. **Войдите в режим сопряжения** (Home + Trigger 5 сек)
3. **Проверьте светодиод** - должен мигать синим

### Шаг 2: Подключение через Serial Monitor

Откройте Serial Monitor (115200 baud) и введите:

```
gearvr_connect
```

Или через Web Serial UI (settings.html):
- Нажмите кнопку **"Connect Gear VR"**

## Troubleshooting

### Проблема: "Could not connect to controller"

**Причины**:
1. Контроллер не в режиме сопряжения
2. Батарея разряжена
3. Слишком далеко от ESP32-S3
4. WiFi включен (создаёт помехи BLE)

**Решение**:
```
✗ FAILED: Could not connect to controller
  → Check controller is powered on
  → Press Home + Trigger for 5 sec (pairing mode)
  → Move controller closer (<5m)
```

1. Проверьте батарейку CR2032
2. Войдите в режим сопряжения (Home + Trigger 5 сек)
3. Поднесите контроллер ближе (<2 метра)
4. Убедитесь, что WiFi отключен: `WiFi.mode(WIFI_OFF)`

---

### Проблема: "Oculus service not found"

**Причины**:
1. Подключен не тот контроллер (не Gear VR)
2. MAC адрес неправильный
3. Прошивка контроллера несовместима

**Решение**:
```
✗ FAILED: Oculus service not found!
  → Wrong device or incompatible firmware
```

1. Проверьте MAC адрес контроллера
2. Убедитесь, что это **Samsung Gear VR Controller** (SM-R324 или SM-R325)
3. Попробуйте другой контроллер

---

### Проблема: Контроллер отключается через несколько секунд

**Причины**:
1. Слабый сигнал BLE (RSSI < -70 dBm)
2. Помехи от WiFi или других устройств
3. Батарея разряжена

**Решение**:
1. Поднесите контроллер ближе
2. Отключите WiFi: `WiFi.mode(WIFI_OFF)`
3. Замените батарейку CR2032
4. Проверьте RSSI в логах (должен быть > -60 dBm)

---

### Проблема: Данные не приходят (touchpad, buttons)

**Причины**:
1. Не отправлена команда инициализации
2. Подписка на уведомления не сработала

**Решение**:
1. Проверьте лог:
   ```
   ✓ Subscribed to data stream (60-byte packets)
   ✓ Sent enable sensors command
   ```
2. Если этих строк нет - переподключите контроллер
3. Отключите и подключите заново:
   ```
   gearvr_disconnect
   gearvr_connect
   ```

## Автоподключение

Чтобы контроллер подключался автоматически при старте ESP32-S3:

**В `setup()` раскомментируйте**:
```cpp
// Gear VR Controller init (BLE)
gearvr_init();
gearvr_connect();  // ← Раскомментировать эту строку
```

**Важно**: Контроллер должен быть включен и в режиме сопряжения при старте ESP32-S3.

## Режим энергосбережения

Контроллер автоматически выключается через **3 минуты** бездействия.

**Как разбудить**:
1. Нажмите любую кнопку
2. Контроллер автоматически переподключится (если `gearvr_update()` работает)

**Принудительное выключение**:
- Зажмите **Home** на 10 секунд

## Технические детали

### BLE Параметры

```cpp
Connection Interval: 12ms (низкая задержка)
Slave Latency: 0
Supervision Timeout: 510ms
MTU: 23 bytes (default)
```

### Частота обновления

```cpp
Data Stream: ~60 Hz (60-byte packets)
Battery Check: Every 60 seconds
Reconnect Timeout: 5 seconds
```

### RSSI (Signal Strength)

| RSSI | Качество | Расстояние |
|------|----------|------------|
| -30 to -50 dBm | Отлично | <1m |
| -50 to -60 dBm | Хорошо | 1-3m |
| -60 to -70 dBm | Средне | 3-5m |
| -70 to -80 dBm | Плохо | 5-8m |
| < -80 dBm | Очень плохо | >8m |

## FAQ

**Q: Нужно ли вводить контроллер в режим сопряжения каждый раз?**

A: Нет, только при первом подключении или после сброса. После успешного подключения контроллер запомнит ESP32-S3.

**Q: Можно ли подключить несколько контроллеров?**

A: Текущая реализация поддерживает только один контроллер. Для нескольких нужно модифицировать код.

**Q: Работает ли контроллер с телефоном одновременно?**

A: Нет, контроллер может быть подключен только к одному устройству. Нужно отключить от телефона перед подключением к ESP32-S3.

**Q: Как узнать MAC адрес моего контроллера?**

A: 
1. Подключите к Android телефону
2. Откройте Settings → Bluetooth → Paired devices
3. Найдите "Gear VR Controller" и посмотрите MAC адрес
4. Или используйте BLE Scanner app

**Q: Контроллер подключается, но через 30 секунд отключается**

A: Это нормально, если не отправлена команда инициализации. Проверьте, что в логах есть:
```
✓ Sent enable sensors command
```
